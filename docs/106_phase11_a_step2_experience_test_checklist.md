# 106 — Phase11-A Step2 体験テスト用チェックリスト

**目的**: 実際に使いながら文言の違和感を洗い出し、文言改善の観察記録に使う。

**前提**: 機能変更は行わない。Phase11-A Step1 適用後の表示文言を前提とする。

**使い方**: テスト日・担当を記入し、該当する項目にチェックまたはメモする。気づいた点は「改善メモ」に随時追記する。

## 1. 毎日触る操作

以下を実際に操作したうえで、文言や流れに違和感がなかったかチェックする。

### 1.1 ノード選択

- [ ] **一覧表示**：「フラット」「ツリー」の切り替えで、どちらが「今の表示」か一瞬で分かったか
- [ ] **未選択時**：「左の一覧からノードをクリックしてください」で、次にやることが伝わったか
- [ ] **ノードクリック後**：詳細パネルの「状態」「温度」「途中内容」「更新」のラベルで、意味が取れたか
- [ ] **ツリーで「子 n 件」**：開閉や件数表示で迷わなかったか

**メモ**:

### 1.2 構成案の生成・反映（提案パネル「構成案」タブ）

- [ ] **タブ「構成案」**：Organizer の機能だと直感できたか／「構成案」という言葉に違和感はないか
- [ ] **「構成案を生成」**：ボタンで何が起きるか分かったか
- [ ] **「反映できる変更案（relation / grouping / decomposition）」**：括弧内の英語と「変更案」の関係が分かったか
- [ ] **「この変更を反映する」**：押すと何が起きるか分かったか
- [ ] **「反映中…」**：待ち時間の説明として自然だったか
- [ ] **理由（任意）・「適用の理由を入力」**：入力するかどうか迷わなかったか

**メモ**:

### 1.3 反映した変更の確認（提案パネル「構成案」タブ内・履歴ブロック）

- [ ] **「反映した変更の履歴」**：見出しの意味がすぐ分かったか
- [ ] **種別「すべて / 関連 / グループ化 / 分解」**：絞りたいときに選びやすかったか
- [ ] **「絞り込む」「クリア」**：操作意図が伝わったか
- [ ] **0件時「まだ反映した変更はありません」**：初回や空状態として自然だったか
- [ ] **「この変更を候補に再表示する」**：履歴から再度反映候補に戻す操作だと理解できたか
- [ ] **「再表示した変更案」**（履歴から復元したカード）：「候補に再表示」との違いで迷わなかったか

**メモ**:

### 1.4 判断案の利用（提案パネル「判断案」タブ）

- [ ] **タブ「判断案」**：Advisor の機能だと直感できたか／「判断案」という言葉に違和感はないか
- [ ] **「判断案を生成」**：ボタンで何が起きるか分かったか
- [ ] **「対象 Node（未指定なら 1 件目）」**：どれが対象か分かったか
- [ ] **「選択中の案」**：今選んでいる案がどれか分かったか
- [ ] **「状態を変更する」**（セクション見出し）：ステータス変更であることが伝わったか
- [ ] **「反映する」**（ボタン）：「この状態にする」と混同しなかったか
- [ ] **「選択を解除」**：案を外す操作だと分かったか

**メモ**:

### 1.5 ダッシュボード上の状態変更（「何が起きた？」〜推定）

- [ ] **「何が起きた？」**：状態のメモ／推定の入口だと分かったか
- [ ] **「推定する」**：キーワードから状態を推定する操作だと分かったか
- [ ] **「この状態にする」**：候補の状態へ遷移する操作だと分かったか
- [ ] **「反映中…」**：ダッシュボード側の反映待ちとして自然だったか
- [ ] **「関連する直近履歴」**：直近の変更履歴であることが伝わったか

## 2. エラー発生時の心理的印象

エラーが出たとき（または意図的にエラーを再現したとき）、以下の観点でメモする。

- [ ] **「反映に失敗しました」**：何が失敗したか想像しやすかったか／不安や苛立ちを感じたか
- [ ] **「確認の取得に失敗しました」**：技術的な印象が強くないか／次のアクションが思い浮かんだか
- [ ] **「通信に失敗しました。ネットワークを確認して再実行してください」**：対処が分かったか
- [ ] **「ステータス変更に失敗しました」**：どの操作の失敗か分かったか
- [ ] **「反映は成功しましたが、画面更新に失敗しました。再読み込みしてください」**：状況と次の一手が分かったか
- [ ] **「現在の状態が分からないため反映できません」**：なぜボタンが押せないか／何をすればよいか伝わったか
- [ ] **「対象Nodeが見つかりません」**：誰向けのメッセージか／対処が浮かんだか
- [ ] **「UUID形式ではありません」**（node_id フィルタ）：入力ミスだと分かったか

## 3. 「分かりやすいか？」の主観評価

5段階でなくてもよい。◎○△×や短いコメントでよい。

| 観点 | 評価・コメント |
|------|----------------|
| 初めて開いたとき、どこから触ればよいか分かったか |  |
| 「構成案」「判断案」の違いが言葉だけで伝わったか |  |
| 「反映する」「この変更を反映する」が、何をどうすることか分かったか |  |
| 「反映した変更の履歴」と「関連する直近履歴」の違いが分かったか |  |
| 確認ダイアログ（Node A と B の間に… / グループ化します / 子を n 件作成…）で、実行内容を理解できたか |  |
| 成功メッセージ（「反映しました（…）」等）で、何が起きたか十分分かったか |  |

## 4. 迷う瞬間がないかの確認項目

以下で「一瞬迷った」「どこを押すか探した」があればチェックし、状況をメモする。

- [ ] 構成案を出したあと、どこに「反映」ボタンがあるか探した
- [ ] 履歴の「種別」と「node_id」のどちらを先に使えばよいか迷った
- [ ] 「再表示した変更案」と「反映できる変更案」の違いで迷った
- [ ] 判断案で「反映する」を押す前に、どの状態に変わるか確認したくて探した
- [ ] 「監査用詳細」を開くかどうか迷った（開いて意味が分かったか）
- [ ] トレー（実施中・判断待ち等）の意味と、一覧の関係が一瞬分からなかった
- [ ] 「要確認」バッジの意味が分からなかった
