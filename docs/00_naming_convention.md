# 0 — ドキュメント（.md）の命名規則

## 規則

ドキュメント用 Markdown ファイル（`docs/*.md`）のファイル名は、次の形式に従う。

```
{連番}_{英語スネークケース}.md
```

- **連番**：2 桁の数字（例：`00`, `01`, `42`）。先頭ゼロで桁を揃える。
- **区切り文字**：**アンダースコア 1 文字 `_`** に統一する。
- **英語スネークケース**：小文字の英単語をアンダースコアでつなぐ（例：`naming_convention`, `phase4_e2e_checklist`）。
- **特殊文字**：全角アンダースコア（‗）やその他の特殊文字は使用しない。使用するのは半角英数字と半角アンダースコアのみ。

## 例

| 正しい例 | 誤った例 |
|----------|----------|
| `00_naming_convention.md` | `00‗naming_convention.md`（全角等の特殊文字） |
| `42_phase4_e2e_checklist.md` | `42-phase4-e2e-checklist.md`（ハイフン） |
| `41_phase4_quality_pipeline.md` | `41 Phase4 Quality Pipeline.md`（スペース・大文字） |

## 運用

- 新規に `.md` を追加するときは、この規則に従って命名する。
- AI やツールで md を生成する場合も、必ずこの規則に従う。
- 既存ファイルのリネームは、必要に応じて段階的に行う（参照の更新に注意）。
