# 120 — Phase13 ダークモード UX 検証

Phase12-Dark クローズ後の「実効性検証」を行う。目的は**「動くか」ではなく「価値が上がったか」の確認**である。

**参照**: docs/117_dark_mode_design.md、docs/118_dark_mode_impl_tasks.md、docs/119_phase12_dark_final_check_result.md、docs/114_phase_status.md。

---

## 1. 検証の位置づけ

- **Phase12-Dark**: /dashboard のダークモード実装（Block A〜C）を完了し、MVP 合格・CLOSED とした。
- **Phase13-UX評価**: 導入後の UX・判断速度・疲労軽減が実証できるかを、定性的＋定量的に確認する。
- **成果の使い方**: 本ドキュメントに検証結果と改善提案を記録し、Phase13 の DONE 判断および今後のテーマ拡張（全ページ・high-contrast 等）の根拠とする。

---

## 2. 検証項目（定性的＋定量的）

### (1) 判断速度

| 観点 | 確認内容 | 結果メモ（実施後に記入） |
|------|----------|---------------------------|
| 危険表示への気づき | 危険表示（エラー・異常検知・StatusBadge CANCELLED 等）に気づくまでの時間・体感 |  |
| 状態ラベルの認識 | StatusBadge（DONE / 要注目 / 中性）の識別がすぐできるか |  |
| 大賢者ブロックの誤認 | 大賢者ブロックが「警告」と誤認されないか（sage と warning の区別） |  |

### (2) 目の疲労感（主観評価）

| 観点 | 確認内容 | 結果メモ（実施後に記入） |
|------|----------|---------------------------|
| 15 分使用後の疲労感 | ライト・ダークそれぞれで約 15 分使用した後の目の疲れの主観評価（例: 5 段階） |  |
| ライト/ダークの比較 | どちらが長時間見やすいか、個人の好みと実感 |  |

### (3) 読みやすさ

| 観点 | 確認内容 | 結果メモ（実施後に記入） |
|------|----------|---------------------------|
| 長文の可読性 | 履歴・提案パネル内の長文が読みやすいか |  |
| muted テキストの視認性 | --text-muted の補足文が「読めない」ほど弱くないか |  |

### (4) 操作誤認

| 観点 | 確認内容 | 結果メモ（実施後に記入） |
|------|----------|---------------------------|
| ボタンと非ボタンの識別 | クリック可能なボタンとラベル・枠の見分けがつくか |  |
| disabled の誤クリック | 無効ボタンが「押せそう」に見えず、誤クリックしにくいか |  |

### (5) 想定外の視認性問題

| 観点 | 確認内容 | 結果メモ（実施後に記入） |
|------|----------|---------------------------|
| bg-sage 上の文字 | 大賢者ブロック（--bg-sage）上の --text-sage が十分読めるか |  |
| warning / danger の区別 | 警告（黄系）と危険（赤系）が混同されないか |  |
| ダーク時のコントラスト低下 | 特定のブロック・文字で「沈んだ」「見づらい」箇所の有無 |  |

---

## 3. 実施方法

- **自身で 20 分使用**: /dashboard でライト・ダーク・システムを切り替えながら、一覧選択・推定フロー・ProposalPanel・Observer・大賢者ブロック等を実際に操作する。上記 (1)〜(5) を意識して観察する。
- **可能なら他者 1 名に触ってもらう**: 同じ項目を短時間でもよいので触れてもらい、第一印象・違和感を聞く。
- **気づいた違和感を全て列挙**: 改善前提で、小さな違和感も含めて列挙する。本ドキュメントの「結果メモ」および「改善提案」に記載する。
