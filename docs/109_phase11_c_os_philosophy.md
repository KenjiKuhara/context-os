# 109 — Phase11-C OS 思想定義と人格接続

本ドキュメントは Phase11-C の前提として、本 OS を「思考で整え、行動で完了させる行動OS」と定義し、その思想に基づく原則と大賢者型人格との接続を定める。

**参照**: [107_phase11_c_personality_design.md](107_phase11_c_personality_design.md)（大賢者型・秘書人格の具体設計）

---

## 1. OS 思想定義書（1000字以内）

本 OS は**「思考で整え、行動で完了させる行動OS」**である。

思考は、タスクの捕捉・言語化・優先づけ・関係づけを通じて「机の上」を整える。状態という共通言語で、やるべきこととその関係を可視化する。ここで完結させるのではなく、思考の成果はつねに**次の一手（行動）**へ渡す設計とする。

行動は、選択・反映・状態変更・完了という形で、思考が整えたものを現実に閉じる。未完了のまま滞留させるのではなく、判断して動かし、必要なら分割・委任・中止まで含めて「決着」をつける。行動の結果は履歴として残り、次の思考の材料になる。

したがって本 OS の役割は、**思考の段階では整理と可視化を支え、行動の段階では判断の最適化と実行の後押しをすること**である。タスクを「ためる」のではなく「流す」ことを前提に、滞留・未決定・長期停滞を検知したときには、事実に基づいて次の行動を提示する。感情で励ましたり咎めたりはせず、状態と選択肢だけを簡潔に示し、決定権は常にユーザーに残す。

この思想を表示層で体現するのが、重要局面のみに現れる大賢者型・秘書人格である。通常表示では人格を出さず、判断や行動の要所でのみ「マスター」と呼び、最適行動の推奨を差し出す。

---

## 2. 行動優先原則 5 つ

行動を軸に設計・表示を揃えるための五原則。

| # | 原則 | 内容 |
|---|------|------|
| 1 | **行動で閉じる** | 思考の成果は、可能なかぎり「反映する」「状態を変更する」「完了する」などの行動で閉じる。ためたままにしない。 |
| 2 | **滞留を可視化する** | 判断待ち・実施中の長期停滞・未着手の蓄積は、件数・経過時間などの事実で示す。隠さず、次の行動候補に結びつける。 |
| 3 | **一手を明確にする** | 現状から導かれる「次の一手」を、推奨として短く提示する。複数ある場合は選択肢を並べ、ユーザーが選ぶ。 |
| 4 | **履歴を行動の証とする** | 反映した変更・状態変更は履歴に残す。振り返りと説明可能性を確保し、次の思考の入力とする。 |
| 5 | **人格は行動の補佐に限定する** | 重要局面でのみ人格（マスター呼称・推奨）を出す。日常の一覧・ラベルは無人格とし、行動に集中させる。 |

---

## 3. 判断最適化原則 5 つ

判断の質とタイミングを支える五原則。

| # | 原則 | 内容 |
|---|------|------|
| 1 | **事実で示す** | 状態・件数・経過時間・関係性など、観測可能な事実だけを提示する。憶測や感情は判断材料に含めない。 |
| 2 | **選択肢を整理する** | 判断すべきポイントでは、取りうる選択肢とその結果を簡潔に整理する。選びやすくし、押し付けない。 |
| 3 | **理由を短く付与する** | 推奨には「滞留が長いため」「期限が近いため」など、理由を一言付与する。根拠のない断定は避ける。 |
| 4 | **取り消し困難な操作は確認する** | 関係追加・グループ化・分解など、一度行うと戻しにくい操作の前には、内容を確認する。 |
| 5 | **判断の先送りを検知する** | 判断待ちの増加・最古の更新からの経過時間などを検知し、要所でのみ「優先順位の確認を推奨します」などと提示する。 |

## 4. 大賢者型人格との接続設計

### 4.1 思想との対応

| OS 思想・原則 | 大賢者型人格での実現（107 対応） |
|---------------|----------------------------------|
| 思考で整え、行動で完了させる | 通常は無人格で「整え」の UI を提供。重要局面で「完了」に近づける一手を推奨。 |
| 行動優先・一手を明確にする | 1.2 最適行動の提示。重要局面メッセージで「〜を推奨します」「次の操作を選んでください」と差し出す。 |
| 判断最適化・事実で示す | 1.2 事実ベース。1.5 許容する表現（事実の提示・推奨・理由の簡潔な付与）。 |
| 人格は行動の補佐に限定 | 1.3 常時は中立・無人格。重要局面のみマスター呼称。1.4 人格の濫用禁止。 |
| 判断の先送りを検知 | 107 §2 出現条件 C1（判断待ちの滞留）・C2（実施中の長期停滞）・C3（未着手の蓄積）等。 |

### 4.2 原則からメッセージ設計への落とし込み

- **行動優先原則** → 重要局面メッセージは「次の行動」を主語にする。「〜があります」で終わらせず、「〜を推奨します」「〜してください」で行動を促す。107 §3 の 50 例はこの方針に沿ってある。
- **判断最適化原則** → メッセージは事実（件数・時間・状態）を先に述べ、そのうえで推奨と理由を短く付ける。確認ダイアログでは選択肢と結果を整理し、取り消し困難な操作は 107 C5 で確認を表示する。

### 4.3 更新方針

- 107（人格設計書・出現条件・メッセージ例・UI 統合方針）は、本 109 の思想・原則に従う**具体化**として位置づける。
- 今後の文言追加・変更では、行動優先原則 5 つと判断最適化原則 5 つに照らし、ずれがないかを確認する。思想と人格が一貫した「行動OS」として振る舞うことを目標とする。

---

以上。OS 思想定義・行動優先原則・判断最適化原則・大賢者型人格との接続を定義した。
